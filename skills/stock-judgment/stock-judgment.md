# 銘柄判定スキル

日本株・米国株を対象にした「バリュー株 + 変化の継続性」視点の総合投資分析を行うスキル。MCP ツール群・適時開示・Web 調査を統合して、判定結果を Markdown レポートと構造化スコアとして出力する。

## トリガー例

- 「トヨタを判定して」
- 「7203.T の分析をして」
- 「ソニーは今買い時？」
- 「この銘柄の投資判断をして」
- 「NVIDIA の投資判断をお願い」

## 分析の位置付け

日本株・米国株を対象にした「バリュー株 + 変化の継続性」視点の総合分析。単なるバリュエーション比較ではなく、定量データ（PER/PBR/YoY）と定性情報（経営コメント、外部環境、競合構造）を統合して、投資判断として意思決定可能な形にまとめる。

## 投資哲学（5本柱）

1. **バリュー株であること** — PER/PBR が低く、本来価値に対して割安であること。ネットキャッシュ比率もセーフティネットとして評価。
2. **良い変化の継続性と新規性** — 業績 YoY の勢いと経営コメントのトーン変化の両面から、「一時的な上振れ」ではなく「構造的な好転」かを見る。
3. **外部環境分析（PESTEL）** — Political（政治）、Economic（経済）、Social（社会）、Technological（技術）、Environmental（環境）、Legal（規制）の 6 軸で、事業への影響を具体化。
4. **事業構造の理解** — 需要ドライバー（何が売上を伸ばすか）× コストドライバー（何が利益を圧迫するか）の 2 軸で、業績の感応度を把握。
5. **競争環境分析（Five Forces 視点）** — 既存競合、新規参入、代替品、買い手の交渉力、供給者の交渉力から、競争優位の持続性を評価。

## 使用する MCP ツール

| ツール | 用途 |
|---|---|
| `market_quote(symbol)` | 現在株価、PER、PBR、配当利回り、52 週レンジ |
| `market_fundamentals(symbol)` | ROE、ネットキャッシュ比率、業績推移（年次/四半期）、YoY 変化率、セクター/業種 |
| `market_history(symbol, period1, period2?, interval?)` | 直近の株価推移 |
| `market_disclosures(symbol)` | TDnet 適時開示（決算短信、業績修正等） |
| `judgment_get(symbol)` | 過去の判定履歴（比較用） |
| `judgment_save(symbol, result)` | 判定結果を保存 |
| `WebSearch` | 競合・外部環境・業界動向の調査（組み込みツール） |
| `WebFetch` | 開示書類 URL 本文取得（組み込みツール） |

## ワークフロー

### 1. 基本情報取得

`market_quote` と `market_fundamentals` を並列実行して、以下を把握する:

- 現在株価、時価総額、52 週レンジ
- PER、PBR、配当利回り、ROE
- ネットキャッシュ比率
- セクター/業種
- 売上・利益の年次・四半期推移、YoY 変化率

### 2. 履歴確認

`judgment_get(symbol)` で過去の判定履歴を取得。

- 前回判定からの変化（スコア差分、論点差分）を把握
- 前回「様子見」だった銘柄が「買い検討」に昇格する条件が満たされたか等を確認

### 3. 適時開示チェック

`market_disclosures(symbol)` で直近の TDnet 開示を取得。

- **`[revision]` タグが付いたもの（業績修正）は最優先で WebFetch で本文取得**
- 決算短信も WebFetch で取得し、経営コメントのトーン変化を確認
- 中期経営計画、業務提携、M&A 関連の開示は必ず内容を読む

### 4. 外部調査（WebSearch）

以下を重点調査する:

- **競合比較**: `「{企業名} 競合」`、`「{業界} シェア」`、`「{業界} ランキング」`
- **コスト動向**: `「{原材料} 価格推移」`、`「{業界} コスト構造」`
- **規制・政策**: `「{業界} 規制」`、`「{進出国} 政策変更」`、`「{業界} 補助金」`
- **需要動向**: `「{業界} 需要 市場規模」`、`「{製品} 出荷台数」`
- **技術トレンド**: `「{業界} 新技術」`、`「{製品} 代替技術」`

必要に応じて該当記事を `WebFetch` で深掘りする。

### 5. 統合判断

全データから判定を下す:

- バリュエーション（valuation）、変化の継続性（changeContinuity）、成長・利益見通し（growthOutlook）の 3 軸で 0-100 のスコア付け
- 判定ラベル（verdict）: 「買い検討」/「注目」/「様子見」/「見送り」から選択
- **`scoring-criteria.md` で詳細な判断基準を必ず参照すること**

### 6. 出力生成

下記「出力フォーマット」に従った Markdown レポートを作成する。

### 7. 保存

`judgment_save(symbol, result)` で判定結果を保存。次回判定時に `judgment_get` で参照される。

## 出力フォーマット

以下のセクション構成で Markdown レポートを作成する。

### 1. エグゼクティブサマリー

- **判定（verdict）を冒頭に太字で明示**。例: **判定: 注目**
- 3 行程度で核心の論理（なぜその判定か）を提示
- 3 軸スコアのサマリー表を掲載

### 2. バリュエーション分析

以下のテーブルを含める:

| 指標 | 値 | セクター平均 | 評価 |
|---|---|---|---|
| PER | 〇〇倍 | 〇〇倍 | 割安/適正/割高 |
| PBR | 〇〇倍 | 〇〇倍 | 割安/適正/割高 |
| 配当利回り | 〇.〇% | 〇.〇% | 高/中/低 |
| ネットキャッシュ比率 | 〇.〇 | - | 強/中/弱 |
| ROE | 〇.〇% | 〇.〇% | 高/中/低 |

### 3. 業績トレンドと変化の兆候

- **定量**: 売上・営業利益・純利益の YoY 推移（直近 4〜8 四半期）
- **定性**: 決算短信・業績修正の経営コメントの変化
- `[revision]` 開示があれば必ず言及

### 4. 事業構造の分析

- **需要ドライバー**: 何が売上を押し上げるか（数量 × 単価の両面）
- **コストドライバー**: 何が利益を圧迫するか（原材料、人件費、為替等）
- それぞれの感応度（例: 為替 1 円円高で営業利益 〇〇 億円減）

### 5. 競合・業界ポジション

比較テーブルを含める:

| 企業 | 売上規模 | 営業利益率 | PER | PBR | 備考 |
|---|---|---|---|---|---|
| 対象企業 | | | | | |
| 競合 A | | | | | |
| 競合 B | | | | | |

Five Forces 視点の短評も添える。

### 6. 外部環境分析（PESTEL）

| 軸 | 現状 | 対象企業への影響 |
|---|---|---|
| Political | | |
| Economic | | |
| Social | | |
| Technological | | |
| Environmental | | |
| Legal | | |

関連しない軸は「特筆なし」と明記してよいが、省略はしない。

### 7. リスク要因

3 層に分けて列挙:

- **マクロリスク**: 景気後退、金利、為替、地政学
- **業界リスク**: 需要縮小、代替技術、規制強化
- **企業固有リスク**: 訴訟、経営者交代、特定顧客依存

### 8. 投資判断

- **判定（verdict）**: 「買い検討」/「注目」/「様子見」/「見送り」
- **論理の流れ**: バリュエーション → 変化の兆候 → 競合優位 → 外部環境 → 判定、の順で結論までの筋道を示す
- **条件変化シナリオ**: 「〇〇になれば格上げ」「〇〇になれば格下げ」を明記

## 構造化スコア

レポート末尾に以下の JSON 形式で構造化スコアを付与する:

```json
{
  "verdict": "買い検討" | "注目" | "様子見" | "見送り",
  "scores": {
    "valuation": 0-100,
    "changeContinuity": 0-100,
    "growthOutlook": 0-100,
    "total": 0-100
  }
}
```

**スコア目安:**

- 90 = 非常に高評価
- 65 = 良好
- 35 = やや懸念
- 10 = 大きな懸念

**詳細な判断基準は `scoring-criteria.md` を参照。**

## 分析上の注意点

- **数値は必ずファクト（MCP ツールまたは WebSearch から取得）に基づく**。推測値を混ぜない。
- **表面的マクロトレンドに飛びつかない**。「AI ブームだから」ではなく、個別企業の製品がどの AI バリューチェーンのどこで売れているかまで具体化する。
- **一時的要因と構造的要因を区別する**。為替差益、特需（例: 半導体バブル）を業績の継続要因と混同しない。
- **都合の良い情報に偏らない**。強気材料と弱気材料の両面を必ず列挙し、リスク要因を過小評価しない。
- **前回判定との整合性**: 「様子見 → 買い検討」に昇格する場合は何が変わったかを明示する。
- **`scoring-criteria.md` で詳細な判断基準を必ず参照**。
