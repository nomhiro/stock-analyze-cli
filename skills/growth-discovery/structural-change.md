# パイプライン1: 構造変化検出（Where to Look）

不可逆的なマクロトレンドを検出し、投資テーマとして構造化する。Peter Lynch の「社会・技術・規制の地殻変動から投資アイデアを見つける」、Clayton Christensen の「破壊的イノベーション理論」を土台とする。

## 基本原則

- **不可逆的変化のみを対象とする** — 一時的ブーム、循環的変動、単なる話題性は除外
- **事実抽出と分類に徹する** — LLM は解釈・予想ではなく、複数ソースからの事実の構造化に集中
- **時系列変化量を重視** — 絶対値ではなく、「昨年比で政策予算がX倍」「論文引用が過去3年でY倍」等の差分
- **複数ソースで裏付ける** — 単一記事の主張は採用しない。政府文書・学術論文・業界レポート・複数メディアで突合

## 分析フレームワーク

### 1. 政策・規制不可逆性

各国政府の動向が「一時的な目玉政策」か「超党派・国際合意レベルの不可逆トレンド」かを判定する。

- **政府予算配分の推移**（前年比、中期計画での増額）
- **法規制の立法プロセス**（成立済み / 審議中 / 構想段階）
- **通商政策**（輸出規制、関税、経済安全保障）
- **補助金・税制優遇**（金額規模、対象業種、期間）

判定基準: 超党派合意 / 国際条約 / 複数国で同方向 = 高い不可逆性

### 2. 技術のS字カーブ評価

対象技術が採用曲線（S-curve）のどのフェーズにあるかを判定する。

| フェーズ | 特徴 | 投資妙味 |
|---------|------|----------|
| emergence（黎明期） | 研究段階、商用化前、PoC中心 | 高リスク・高リターン、失敗確率大 |
| growth（成長期） | 商用化開始、急速採用、設備投資拡大 | バランス型、勝者が見え始める |
| maturity（成熟期） | 普及完了、差別化困難、価格競争 | 低成長、配当重視 |
| decline（衰退期） | 次世代技術に置換 | 基本的に回避 |

判定指標: 学術論文数の推移、特許出願トレンド、商用導入事例の数、主要企業の設備投資額

### 3. 市場波及度

テーマが単一業種に閉じるか、複数産業に波及するかを評価する。

- **産業の幅** — テーマが影響を与える業種数
- **サプライチェーンの長さ** — 上流（素材・装置）から下流（サービス）までの段数
- **雇用・消費への影響** — GDP 寄与度、雇用創出規模
- **代替される旧産業の規模** — 破壊されるレガシーの大きさ

## ワークフロー

1. **最新動向のリサーチ**
   - `WebSearch` で過去12ヶ月の政策・規制・産業動向を調査
   - クエリ例: 「2026 semiconductor CHIPS Act budget」「EU AI Act implementation」「日本 GX 推進法 予算」
   - 政府発表、業界団体レポート、主要メディア記事を収集

2. **学術論文トレンドの確認**
   - `market_paper_search` で関連分野の論文を検索
   - 引用数バースト（過去3年で急増）、新規著者数、主要会議採択数を確認
   - 論文から商用化までのタイムラグ（通常5-10年）を考慮

3. **複数ソース突合**
   - 政策 × 学術 × 産業動向が同じ方向を示しているテーマを抽出
   - 矛盾ソースは排除、または「注意点」として記録
   - 一次ソース（政府文書、企業IR、査読論文）を優先

4. **不可逆性スコア算出**
   - 政策後押し度（policySupport）: 法制化状況、予算規模、国際合意の有無 — 0-100
   - 技術成熟度（technologyMaturity）: S字カーブフェーズ、商用化事例数 — 0-100
   - 市場波及度（marketPenetration）: 波及産業数、TAM 予測 — 0-100
   - **総合スコア = 3軸の幾何平均**（一軸でも極端に低いテーマは除外するため加算平均は使わない）

5. **テーマ構造化**
   - 3-7 個のテーマに絞り込む（多すぎると絞り込み効果が薄れる）
   - 各テーマに英語スラグ ID、日本語名、説明（200-400字）を付与
   - 関連セクター、時間軸（medium: 3-5年 / long: 5-10年）を明記

## 出力（JSON）

```json
{
  "themes": [
    {
      "id": "テーマID（英語スラグ）",
      "name": "テーマ名",
      "description": "200-400文字",
      "irreversibilityScore": 0-100,
      "scores": {
        "policySupport": 0-100,
        "technologyMaturity": 0-100,
        "marketPenetration": 0-100
      },
      "evidence": [
        { "source": "...", "summary": "...", "url": "..." }
      ],
      "sCurvePhase": "emergence | growth | maturity | decline",
      "timeHorizon": "medium | long",
      "relatedSectors": ["..."]
    }
  ]
}
```

## 注意点

- 不可逆性スコアが 60 未満のテーマは出力に含めない（ノイズ除去）
- evidence は最低 3 つ、できれば一次ソースを含める
- description では「なぜ不可逆か」を端的に言語化する
- 関連セクターは東証業種コード / GICS セクター名で記述する
