# 銘柄判定スキル

日本株・米国株を対象にした「バリュー株 + 変化の継続性」視点の総合投資分析を行うスキル。MCP ツール群・適時開示・Web 調査を統合して、判定結果を Markdown レポートと構造化スコアとして出力する。

## トリガー例

- 「トヨタを判定して」
- 「7203.T の分析をして」
- 「ソニーは今買い時？」
- 「この銘柄の投資判断をして」
- 「NVIDIA の投資判断をお願い」

## 分析の位置付け

日本株・米国株を対象にした「バリュー株 + 変化の継続性」視点の総合分析。単なるバリュエーション比較ではなく、定量データ（PER/PBR/YoY）と定性情報（経営コメント、外部環境、競合構造）を統合して、投資判断として意思決定可能な形にまとめる。

## 投資哲学（5本柱）

1. **バリュー株であること** — PER/PBR が低く、本来価値に対して割安であること。ネットキャッシュ比率もセーフティネットとして評価。
2. **良い変化の継続性と新規性** — 業績 YoY の勢いと経営コメントのトーン変化の両面から、「一時的な上振れ」ではなく「構造的な好転」かを見る。
3. **外部環境分析（PESTEL）** — Political（政治）、Economic（経済）、Social（社会）、Technological（技術）、Environmental（環境）、Legal（規制）の 6 軸で、事業への影響を具体化。
4. **事業構造の理解** — 需要ドライバー（何が売上を伸ばすか）× コストドライバー（何が利益を圧迫するか）の 2 軸で、業績の感応度を把握。
5. **競争環境分析（Five Forces 視点）** — 既存競合、新規参入、代替品、買い手の交渉力、供給者の交渉力から、競争優位の持続性を評価。

## 使用する MCP ツール

### MCP サーバー未接続時のフォールバック

Claude Code セッションに `stock-analyze-cli` の MCP サーバーが接続されていない場合、`mcp-server/src/lib/` のライブラリを直接呼ぶワンショット TypeScript スクリプトを書いて `npx tsx` で実行する。

```ts
// scripts/analyze-{symbol}.ts
import { getQuote, getStockFundamentals, getHistory } from "../mcp-server/src/lib/yahoo-finance";
import { getDisclosuresBySymbol } from "../mcp-server/src/lib/tdnet";

async function main() {
  const symbol = "1434.T";
  const [q, f, h, d] = await Promise.all([
    getQuote(symbol),
    getStockFundamentals(symbol),
    getHistory(symbol, "2025-04-01", "2026-04-30", "1mo"),
    getDisclosuresBySymbol(symbol, 60),
  ]);
  console.log(JSON.stringify({ q, f, h, d }, null, 2));
}
main();
```

## Web データ取得の使い分け（重要）

`WebFetch` だけに頼ると、PDF パース失敗・User-Agent ブロック・WAF challenge で詰まることが多い。**失敗パターンに応じて取得手段を切り替える**。

### 失敗パターン別の対処

| 症状 | 原因 | 最適手段 |
|---|---|---|
| WebFetch がバイナリ PDF を返してパース失敗 | WebFetch 内蔵 PDF→テキスト変換が CJK 埋込フォントで失敗 | `curl -A "<browser-UA>" -o file.pdf <url>` → `pdftotext -enc UTF-8 -layout file.pdf -` |
| TDnet PDF が 404 | TDnet は約30日で過去 PDF を削除 | 同 PDF を発行元 IR サイトから取得（例: `{company-domain}/ja/news/ir/auto_<開示ID>/pdfFile.pdf`） |
| HTTP 403 が即返る（kabutan / minkabu / 株探など） | デフォルト User-Agent によるボットブロック | `curl -A "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 ..."` |
| HTTP 202 で `x-amzn-waf-action: challenge` ヘッダー（buffett-code 等） | AWS WAF JavaScript challenge | **Playwright MCP** で `browser_navigate` → `browser_evaluate(() => document.body.innerText)` |
| Cloudflare の "Just a moment..." ページ | Cloudflare bot challenge | 同上（Playwright / chrome-devtools MCP） |

### 診断手順

```bash
# まず HTTP ステータスとヘッダーで診断
curl -sI -A "Mozilla/5.0 ..." "<url>" | head -20
# 200 → そのまま GET（curl -A ... | head）で本文取得
# 403 → User-Agent 不足 or 別の制限
# 202 + x-amzn-waf-action: challenge → Playwright 必須
# 404 → URL 間違い or 期限切れ（TDnet なら IR サイトへ切替）
```

### PDF 一括取得→テキスト化のテンプレ

```bash
mkdir -p tmp/{symbol}
UA="Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/130.0.0.0 Safari/537.36"
# 並列ダウンロード → pdftotext で UTF-8 抽出
for url in "<url1>" "<url2>"; do
  fname=$(basename "$url")
  curl -sL -A "$UA" "$url" -o "tmp/{symbol}/$fname"
done
for f in tmp/{symbol}/*.pdf; do
  pdftotext -enc UTF-8 -layout "$f" "${f%.pdf}.txt"
done
```

抽出後は Read ツールで読む。全PDF（決算短信・決算説明会資料・中期経営計画）を取れば、セグメント別利益・受注高・セグメント計画など WebSearch では拾えない一次情報が手に入る。

### Playwright MCP の使い方（WAF 突破）

```
browser_navigate(url)          # ページ遷移、WAF challenge を JS で通過
browser_evaluate(() => {
  return document.body.innerText.split('\n').filter(l => l.trim()).slice(0, 200).join('\n');
})
browser_close()                 # 必ず閉じる
```

buffett-code から取得すべき指標: PER（会予）、ROIC、WACC、株主資本コスト、β、PEG（会予）、5年CAGR（予想込）、配当性向。これらは Yahoo Finance では取れない。

## ワークフロー

### 1. 基本情報取得

`market_quote` と `market_fundamentals` を並列実行して、以下を把握する:

- 現在株価、時価総額、52 週レンジ
- PER、PBR、配当利回り、ROE
- ネットキャッシュ比率
- セクター/業種
- 売上・利益の年次・四半期推移、YoY 変化率

### 2. 履歴確認

`judgment_get(symbol)` で過去の判定履歴を取得。

- 前回判定からの変化（スコア差分、論点差分）を把握
- 前回「様子見」だった銘柄が「買い検討」に昇格する条件が満たされたか等を確認

### 3. 適時開示チェック（最重要）

`market_disclosures(symbol)` で直近の TDnet 開示を取得。**決算短信・決算説明会資料・中期経営計画の3点セットを必ず PDF で取得して読む**（WebSearch のサマリーでは見落とす情報が多い）。

- **`[revision]` タグが付いたもの（業績修正）は最優先で WebFetch で本文取得**
- 決算短信は本文中の **セグメント別の売上・利益・受注高を必ず抽出**（後述の利益分解に必須）
- 中期経営計画は **セグメント別の売上・利益の最終年度計画値**を必ず抽出（一過性利益かどうかの判定材料）
- 業績修正、業務提携、M&A 関連の開示も内容を読む

PDF の取得・パースで詰まったら「Web データ取得の使い分け」節に従って `curl + pdftotext` に切り替える。

### 3.5 セグメント利益分解（一過性 vs 経常的）

**反復的に発生しないセグメント利益を切り分けないと、変化の継続性スコアを過大評価する**。

決算短信本文から以下を抽出する:

- **セグメント別 売上・利益**（直近1H/Q + 前年同期 + 通期計画）
- **特殊要因**: 不動産売却、資産売却益、負ののれん発生益、為替差損益、保険金収入、補助金、訴訟和解金 など
- **中計でのセグメント計画値**（最終年度）と現状の乖離

判別ガイドライン:

| 兆候 | 解釈 |
|---|---|
| 営業利益YoYが売上YoYの3倍以上 | 何らかの特殊要因の可能性、必ず内訳を確認 |
| 中計でセグメント利益が「縮小」前提 | 会社が「一過性」と認識している証拠 |
| 1H で通期予想の70%以上を消化 | 上方修正期待だが、内訳が一過性なら下半期は反落 |
| 特定セグメントが営業利益の50%超 | そのセグメントの継続性が判定の核心 |

例: ある銘柄の 1H 営業利益 ¥1,315M のうち、不動産事業 ¥779M（59%）が販売用不動産2件売却（一過性）。中計では同セグメント営業益を 3年で -24% 縮小する計画 → 表面的な YoY +119% は構造成長と直結しない。本業セグメントのみで再計算した利益伸び率を採用する。

### 4. 外部調査（WebSearch + buffett-code）

以下を重点調査する:

- **競合比較**: `「{企業名} 競合」`、`「{業界} シェア」`、`「{業界} ランキング」`
- **コスト動向**: `「{原材料} 価格推移」`、`「{業界} コスト構造」`
- **規制・政策**: `「{業界} 規制」`、`「{進出国} 政策変更」`、`「{業界} 補助金」`
- **需要動向**: `「{業界} 需要 市場規模」`、`「{製品} 出荷台数」`
- **技術トレンド**: `「{業界} 新技術」`、`「{製品} 代替技術」`
- **会社予想ベース指標**: buffett-code で **PER（会予）／ROIC／WACC／株主資本コスト／β／PEG（会予）／5年CAGR（予想込み）／配当性向** を取得（Yahoo Finance では取れない）

必要に応じて該当記事を `WebFetch` で深掘りする。403/WAF で詰まったら `curl + browser-UA` か Playwright MCP に切り替える。

#### 受注高・受注残の確認（先行指標）

決算短信本文から以下を必ず抽出（特に建設業・受注産業）:

- **当期受注高 YoY**（売上 YoY より先行）
- **繰越受注残高の期首→中間期末 推移**（売上の継続性を裏付け）

受注高 YoY が売上 YoY を大きく上回っているなら、下半期〜次期の売上加速が見込める。受注残が縮小しているなら逆。

#### 競合バリュエーション一括取得

同業他社の PER/PBR/ROE は、`scripts/{symbol}-peers.ts` を作って `getQuote` + `getStockFundamentals` を `for` ループで呼べば一括取得できる（30秒程度）。WebSearch では PER/PBR の正確な数値が取れないことが多い。

### 5. 統合判断

全データから判定を下す:

- バリュエーション（valuation）、変化の継続性（changeContinuity）、成長・利益見通し（growthOutlook）の 3 軸で 0-100 のスコア付け
- 判定ラベル（verdict）: 「買い検討」/「注目」/「様子見」/「見送り」から選択
- **`scoring-criteria.md` で詳細な判断基準を必ず参照すること**

### 6. 出力生成

下記「出力フォーマット」に従った Markdown レポートを作成する。

### 7. 保存

`judgment_save(symbol, result)` で判定結果を保存。次回判定時に `judgment_get` で参照される。

以下のセクション構成で Markdown レポートを作成する。

### 2. バリュエーション分析

以下のテーブルを含める（セクター平均が取得できない指標は同業他社の中央値で代用）:

| 指標 | 値 | 算定基準 | 評価 |
|---|---|---|---|
| 株価 | 〇〇円 | 52週レンジ ¥XX-¥XX の N% 位置 | - |
| 時価総額 | 〇〇億円 | - | - |
| PER (TTM) | 〇〇倍 | EPS ¥XX | 割安/適正/割高 |
| PER (会社予想) | 〇〇倍 | EPS ¥XX | 会社予想保守性込み |
| PBR | 〇〇倍 | BPS ¥XX | 割安/適正/割高 |
| EV/EBITDA | 〇〇倍 | - | - |
| 配当利回り | 〇.〇% | DPS ¥XX | 高/中/低 |
| ネットキャッシュ比率 | 〇.〇 | - | 強/中/弱 |
| ROE | 〇.〇% (TTM) / 〇.〇% (会予) | - | 高/中/低 |
| **ROIC** | 〇.〇% | - | **WACC との対比で価値創造判定** |

| | 直近実績 | 中計最終年度 | 増減率 / CAGR |
|---|---:|---:|---:|
| 全社 売上 | | | |
| 全社 営業利益 | | | |
| ROE | | | |
| 配当性向 | | | |

セグメント別の中計計画値と直近実績の方向性が一致しているかを必ずチェック（縮小前提のセグメントで足元利益が急増しているなら、それは一過性と判定）。

- **需要ドライバー**: 何が売上を押し上げるか（数量 × 単価の両面）

3 層に分けて列挙:

### 8. 投資判断

## 構造化スコア

`judgment_save` の実スキーマに合わせて以下の JSON で保存する:

```json
{
  "verdict": "買い検討" | "注目" | "様子見" | "見送り",
  "scores": {
    "value": 0-100,        // バリュー（割安性）
    "momentum": 0-100,     // 変化の継続性（モメンタム）
    "growth": 0-100,       // 成長見通し
    "total": 0-100         // 総合
  }
}
```

**スコア目安:**

- 90 = 非常に高評価
- 65 = 良好
- 35 = やや懸念
- 10 = 大きな懸念

**詳細な判断基準は `scoring-criteria.md` を参照。**

## 分析上の注意点

- **数値は必ずファクト（MCP ツールまたは公式PDFから取得）に基づく**。推測値を混ぜない。
- **表面的マクロトレンドに飛びつかない**。「AI ブームだから」ではなく、個別企業の製品がどの AI バリューチェーンのどこで売れているかまで具体化する。
- **一時的要因と構造的要因を必ず区別する（最重要）**:
  - 為替差益・特需（半導体バブル等）・不動産売却益・資産売却益・負ののれん発生益・保険金収入・補助金・訴訟和解金は「一過性」
  - **報告 YoY が異常に高い場合（営業利益YoY > 売上YoYの3倍）は必ずセグメント別に分解**
  - **中計でセグメント利益が縮小前提なら、足元の急増は会社自身が一過性と認識している証拠**
  - 一過性を除いた「本業ベース YoY」で momentum スコアを再計算する
- **進捗率の解釈に注意**: 1H で通期予想の70%超を消化していても、その大半が一過性なら上方修正は出ない可能性が高い
- **PER の二重解釈**: TTM ベースで割安に見えても、一過性利益が剥落する翌期予想ベースでは正常水準のことがある
- **WebSearch のサマリーは鵜呑みにしない**。決算短信本文／中計PDFの一次情報を必ず読む
- **都合の良い情報に偏らない**。強気材料と弱気材料の両面を必ず列挙し、リスク要因を過小評価しない
- **前回判定との整合性**: 昇格／降格時は何が変わったかを明示する
- **`scoring-criteria.md` で詳細な判断基準を必ず参照**

## 失敗パターンと反省（過去の教訓）

| 落とし穴 | 兆候 | 対処 |
|---|---|---|
| 一過性利益を構造成長と誤認 | 営業益YoY > 売上YoYの3倍、特定セグメントが利益の50%超 | セグメント別利益分解 + 中計計画値との整合チェック |
| WebFetch のサマリーで完結 | 「決算で増収増益を達成しました」程度の表層情報のみ | 必ず PDF を `curl + pdftotext` で取得し、本文の経営コメント・セグメント別数値・受注高を確認 |
| 同業 PER/PBR を WebSearch で取ろうとして失敗 | 検索結果が古い／曖昧 | `scripts/{symbol}-peers.ts` で MCP ライブラリ直接呼び出し |
| WAF で取れず諦める | HTTP 202、`x-amzn-waf-action: challenge` | Playwright MCP で突破。buffett-code は ROIC/WACC/β 取得に必須 |
| 進捗率高 = 上方修正と短絡 | 1H で通期予想の70%以上消化 | 一過性の有無を確認してから上方修正期待を判断 |
