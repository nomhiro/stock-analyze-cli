# パイプライン3: 市場ギャップ検出（When to Buy）

企業の本質的価値とマーケットの現在評価のギャップを検出し、推奨アクションを判定する。Peter Lynch の「誤解されている優良企業を探せ」、Michael Mauboussin の「市場が織り込んでいる前提を逆算せよ」を土台とする。

## 基本原則

- **市場コンセンサスとの比較に徹する** — 「絶対的に割安/割高」ではなく「市場の予想との差分」を評価
- **ギャップの言語化が最重要** — なぜ市場が誤解しているかを仮説として明示する
- **逆張りチェック（失敗シナリオ3つ）** — 自説に酔わないため、3つの失敗シナリオを必ず生成
- **推奨は3段階のみ** — 「買い/売り」ではなく「詳細分析 / ウォッチリスト / スキップ」に限定

## 市場の誤解パターン

市場コンセンサスと本質価値にギャップが生じる典型パターン。

1. **カテゴリ誤認** — 旧カテゴリとして評価されているが、実態は新カテゴリへ移行中（例: Amazon を書店として評価）
2. **成長率の過小評価** — TAM（Total Addressable Market）拡大が織り込まれていない
3. **堀の過小評価** — スイッチングコストやネットワーク効果の強さが認識されていない
4. **一時的悪材料の過大評価** — 一過性のマイナス要因が長期構造と混同されている
5. **コングロマリットディスカウント** — 複数事業の合算評価で各事業の実力が見えない

## バリュエーション比較視点

価格の妥当性評価は、以下の複数視点を必ず組み合わせる。

- **同業平均との比較** — 同業他社の PER / PBR / EV/EBITDA 中央値
- **PEGレシオ** — PER ÷ 利益成長率（1.0 以下なら割安目安）
- **類似企業水準** — 事業モデルが近い海外企業・前例企業の水準
- **過去レンジ** — 自社の過去5-10年のバリュエーションレンジ内での位置

## 失敗シナリオ生成基準

各企業について3つの失敗シナリオを生成する。各シナリオは以下を満たすこと。

- **具体的条件** — 何が起きれば失敗するかを具体的に記述（抽象的な「競争激化」はNG）
- **確率** — high / medium / low で判定
- **緩和要因** — そのリスクに対して企業側が持っている備え・打ち手
- **モニタリングKPI** — そのシナリオの兆候を早期に掴むための定量指標

## 推奨アクション（3段階）

- **detailed_analysis** — 有力候補。個別に詳細な財務モデリング・経営陣インタビュー調査に進むべき
- **watchlist** — 魅力はあるがタイミング待ち。特定トリガー（決算、製品発表等）を監視
- **skip** — ギャップなし、または失敗シナリオが重い。今回はスキップ

## ワークフロー

1. **市場コンセンサス取得**
   - `WebSearch` で各企業のアナリストコンセンサス・目標株価・将来利益予想を検索
   - クエリ例: 「Nvidia analyst consensus price target 2026」「トヨタ 業績予想 アナリスト」
   - 主要証券会社レポート、ブルームバーグ/ロイター集計、TIKR / Koyfin 等の集計サイト

2. **本質価値との比較**
   - パイプライン2の分析結果（堀、サプライチェーン位置、経営者ビジョン）と市場評価を突き合わせる
   - 「市場は〇〇と見ているが、パイプライン2の分析では××」という差分を抽出

3. **誤解パターンの特定**
   - ギャップがある場合、上記5つの誤解パターンのどれに該当するかを判定
   - 複数該当可だが、最大のものを `hypothesis` として言語化

4. **バリュエーション比較**
   - `currentValuation`: 現状 PER / PBR / EV/EBITDA と市場が織り込むストーリー
   - `fairValuation`: パイプライン2分析を踏まえた妥当水準とその根拠
   - `gap`: undervalued / fairly_valued / overvalued

5. **失敗シナリオ生成（必ず3つ）**
   - 競合・技術・規制・経営・マクロの観点を幅広く網羅
   - 確率と緩和要因を必ずセットで記述
   - モニタリングKPIを明示

6. **推奨アクション判定**
   - gap が undervalued + 失敗シナリオの確率が全て low/medium → `detailed_analysis`
   - gap が undervalued だが失敗シナリオに high が含まれる → `watchlist`
   - gap が fairly_valued or overvalued → `skip`（特段の触媒がない限り）
   - 判定理由を `actionReason` に2-3文で記述

## 出力（JSON）

```json
{
  "companies": [
    {
      "symbol": "...",
      "name": "...",
      "market": "jp | us",
      "marketMisconception": {
        "hypothesis": "市場の誤解仮説",
        "currentValuation": "PER xx倍、○○カテゴリ評価",
        "fairValuation": "PER yy倍が妥当な理由",
        "gap": "undervalued | fairly_valued | overvalued"
      },
      "failureScenarios": [
        { "description": "...", "probability": "high | medium | low", "mitigatingFactor": "..." }
      ],
      "recommendedAction": "detailed_analysis | watchlist | skip",
      "actionReason": "..."
    }
  ]
}
```

## 注意点

- 失敗シナリオは必ず3つ生成する（2つ以下は「分析不足」と解釈される）
- `hypothesis` は単なる一般論（「AI は伸びる」等）ではなく、企業固有の誤解を言語化する
- 推奨アクションに「買い / 売り」は絶対に使わない。投資判断は人間の役割
- アナリストコンセンサスが取得できない企業は、その旨を `actionReason` に明記した上で `watchlist` を上限とする
- バリュエーション指標は同業平均・過去レンジのどちらかを必ず参照点として記載する
